# 🍽️ SchoolMenu — Система за визуализация на дневно меню

Starter проект за производствената практика (X клас).
Направен по документа „Технически бележки — Реализация на проекта“.

**Какво е готово:** входът с роли (кухня/ученик), базата данни (SQLite + EF Core),
работещи примери за запис и четене, и ученическата страница, която показва менюто за днес.

**Какво правиш ти:** Задачи 1–5 (виж по-долу). Всяка задача има стъпки,
описани в коментар `ЗАДАЧА X` в съответния файл.

---

## 🚀 Стартиране

Трябва ти **.NET 8 SDK** (и по желание Visual Studio 2022).

**Вариант А — Visual Studio 2022:** отвори `SchoolMenu.sln` и натисни ▶ (F5).

**Вариант Б — терминал:**

```
cd SchoolMenu.Api
dotnet run
```

Отвори **http://localhost:5000** — трябва да видиш менюто за днес (примерни данни).

При първото стартиране файлът с базата (`menu.db`) **се създава сам** и се пълни
с примерни данни — виж `Data/SeedData.cs`.

## 🔄 Как текат данните (запомни тези две вериги!)

**ЗАПИС** (пример: добавяне на ястие от админ панела):

```
форма в admin.html
  → admin.js събира стойностите
  → api.js postMenuItem()             fetch POST /api/menuitems
  → MenuItemsController.Create()      валидация + _db.Add()
  → _db.SaveChangesAsync()            INSERT в menu.db ✅
```

**ЧЕТЕНЕ** (пример: ученикът отваря страницата):

```
index.html се зарежда
  → student.js loadMenu()
  → api.js getMenuForDate()           fetch GET /api/menu?date=...
  → MenuController.GetByDate()        SELECT + Include (JOIN) от menu.db
  → JSON отговор → student.js го превръща в HTML ✅
```

## 🧪 Swagger — тествай API-то без frontend

В режим Development имаш **http://localhost:5000/swagger** — списък с всички
endpoints, които можеш да пробваш направо от браузъра.

⚠️ За POST/PUT/DELETE първо влез през `login.html` като `kitchen`
**в същия браузър** — иначе получаваш 401.

## 🆘 Чести проблеми

| Проблем | Решение |
|---|---|
| `401 Unauthorized` | Не си влязъл като kitchen — влез през /login.html (в същия браузър) |
| Промених модел, но базата не се промени | Спри приложението, **изтрий `menu.db`**, стартирай пак |
| „Менюто не е въведено“ | Няма меню за тази дата — добави го през админ панела (Задача 1) |
| Порт 5000 е зает | Смени порта в `Properties/launchSettings.json` |
| CORS грешка | Отваряш HTML-а през Live Server — недей! Ползвай http://localhost:5000 |
| `async/await` бърка | Правило: щом вътре има `await`, функцията отпред има `async` |

## 📝 За наставника

- Паролите се хешират с **BCrypt** (`BCrypt.Net-Next`) — никога plain text.
- Базата се създава с `EnsureCreated()` — най-простото за практиката.
  В реален проект се ползват migrations (`dotnet ef migrations add ...`).
- За SQL Server / PostgreSQL се сменя **само един ред** в `Program.cs` (коментиран там).
- `menu.db` е в `.gitignore` — всеки ученик си има локална база.
